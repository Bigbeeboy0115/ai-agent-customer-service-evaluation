# Agent 版本对比与优化建议

## 1. 核心结论

样例数据中，`v3_tool_augmented` 的整体表现最好。它通过知识库查询、风险提醒和转人工规则，显著改善了退款、物流、账号异常和投诉升级等场景。

`v2_prompt_optimized` 相比 `v1_baseline` 已经能明显提升语气、信息完整性和主动追问能力，但在需要系统动作的场景仍然不足，例如物流查询需要查订单，账号异常需要安全队列，投诉升级需要转人工。

`v1_baseline` 的主要问题是回答过短、泛泛而谈、缺少上下文记忆和业务推进。它在高购买意向售前咨询中没有把用户推向试用、优惠或销售跟进，也容易在投诉场景中重复解释规则。

## 2. v1 到 v2 的改进

v2 的主要价值来自 Prompt 优化：

- 回答更结构化，能说明条件、入口、时效和下一步。
- 对负面情绪有一定安抚。
- 对售前问题会反问用户关注点。
- 对高意向用户能做基础套餐推荐。

但 v2 的问题是：它仍然主要依赖文本策略，没有真实工具能力。一旦用户要查物流、查账号状态、处理投诉升级，单靠 Prompt 很难完成闭环。

## 3. v2 到 v3 的改进

v3 的主要价值来自工具增强：

- 知识库补齐退款、发票、物流等政策细节。
- 转人工规则降低投诉和账号异常场景的处理风险。
- 风险提醒避免承诺过度和敏感信息泄露。
- 对高购买意向用户能推进到试用申请、优惠确认或销售顾问跟进。

v3 仍可继续优化的方向是：把不同场景的 Skill 调用成功率单独监控，例如订单查询失败率、知识库命中率、转人工排队时长。

## 4. 场景化优化建议

### 退款咨询

Agent 经常只回答“可以退款”，但没有说明退款条件、时效、入口和费用承担。建议在 Prompt 中加入售后政策回答四要素：条件、时效、入口、费用；当用户情绪为负面时，先安抚再解释规则。

### 高购买意向售前

高购买意向用户已经在问套餐、优惠或试用，但 Agent 如果只回答功能差异，会损失转化机会。建议识别 `purchase_intent_level=high` 后，优先给出套餐差异、适用人群、优惠信息和明确 CTA。

### 投诉升级

投诉场景中，如果用户已经明确表达不满或要求人工，Agent 不应继续重复规则。建议设置转人工规则：当 `escalation_needed=true` 且用户连续两轮负面表达时，必须触发人工客服，并记录升级原因。

### 账号异常

账号异常可能涉及安全风险。Agent 应先提醒用户不要提供密码、验证码等敏感信息，再进入账号安全队列。建议增加账号安全 Skill，并把疑似盗号、验证码失败、登录异常设为高优先级。

### 风险承诺

售前场景不能承诺“一定提升”“保证翻倍”。建议增加安全策略：禁止确定性收益承诺，把回答改为试点验证、A/B 测试和指标观测。

## 5. Analysis Summary

本项目将 AI Agent 的问题拆成业务层和系统层。业务层关注解决率、风险率、转化推进和转人工完成率；系统层进一步定位到 Prompt、Memory、Skill、知识库和安全策略。这样可以把数据结论直接转化为 Agent 迭代建议。
